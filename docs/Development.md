# Development Guide

## Prerequisites

- Python 3.12+
- Docker + Docker Compose (recommended), or a local PostgreSQL 16 + Redis 7
- An MCP Gateway instance reachable at the URL you configure (or a stub
  for local development — see below)

## Local setup without Docker

```bash
python -m venv .venv
source .venv/scripts/activate

pip install -r requirements-dev.txt

cp .env.example .env
# point DATABASE_URL / DATABASE_URL_SYNC at your local Postgres,
# REDIS_URL at your local Redis, MCP_GATEWAY_URL at your Gateway.

alembic upgrade head

uvicorn app.main:app --reload --port 8080
```

## Local setup with Docker Compose

```bash
cp .env.example .env
docker compose up --build
```

Migrations run automatically on container start
(`docker/entrypoint.sh` waits for Postgres, then runs
`alembic upgrade head`, then starts uvicorn).

## Running tests

```bash
pip install -r requirements-dev.txt
pytest -v                      # all tests
pytest tests/unit -v           # fast, no I/O
pytest tests/integration -v    # repository + full API request/response cycle
pytest --cov=app --cov-report=term-missing   # coverage
```

Tests use an in-memory SQLite database via `aiosqlite` — no external
services are required to run the suite. `tests/conftest.py` patches the
PostgreSQL-specific `JSONB` columns to generic `JSON` for the SQLite
engine only; production migrations and the real runtime engine always
use native `JSONB`.

## Database migrations

Migrations are hand-authored under `alembic/versions/` (not
autogenerated against this repo's environment, since no local
PostgreSQL was available during initial authoring — they were
validated by diffing every table's column set against the SQLAlchemy
ORM's PostgreSQL-dialect-compiled DDL, see comments in
`alembic/versions/0001_initial_schema.py`).

To create a new migration after changing a model in `app/models/`:

```bash
alembic revision -m "add some_column to agents"
# edit the generated file under alembic/versions/
alembic upgrade head
```

To autogenerate against a real database once one is available:

```bash
alembic revision --autogenerate -m "description"
# ALWAYS review the generated diff before applying it
alembic upgrade head
```

To roll back:

```bash
alembic downgrade -1
```

## Project conventions

- **No hardcoded agents/prompts/skills/teams/tools.** If you find
  yourself writing a Python literal that represents a specific tenant's
  business data, stop — it belongs in the database via the `/api/v1/*`
  endpoints instead.
- **Repositories never raise HTTP exceptions directly** — they raise
  `app.core.exceptions.AgnoRuntimeError` subclasses (`NotFoundError`,
  `ConflictError`, ...), which `app/main.py`'s exception handler
  translates into the standard error envelope.
- **Services own business logic; routers stay thin.** A router function
  should mostly be: parse request -> call repository/service -> map to
  response schema.
- **Soft delete by default for metadata** (see
  `docs/Architecture.md#9-soft-delete-strategy`). Use
  `repo.soft_delete(obj)`, not `repo.hard_delete(obj)`, unless you have a
  specific reason (memories are the one exception).
- **Never add authorization logic to this codebase.** If you're tempted
  to check a role, scope, or permission anywhere in `app/`, that logic
  belongs in MCP Gateway, not here. See
  `docs/Architecture.md#6-auth-propagation`.

## Mocking MCP Gateway locally

If you don't have a real MCP Gateway running yet, a minimal stub is
enough to exercise the full chat flow:

```python
# mcp_gateway_stub.py
from fastapi import FastAPI, Request

app = FastAPI()

@app.post("/execute")
async def execute(request: Request):
    body = await request.json()
    return {"tool": body["tool"], "result": "ok", "echo": body["arguments"]}

@app.get("/capabilities")
async def capabilities():
    return {"capabilities": [
        {"code": "crm.customer.create", "description": "Create a CRM customer",
         "parameters": {"type": "object", "properties": {"name": {"type": "string"}}, "required": ["name"]}}
    ]}
```

```bash
uvicorn mcp_gateway_stub:app --port 9090
```

Then set `MCP_GATEWAY_URL=http://localhost:9090` in your `.env`.
